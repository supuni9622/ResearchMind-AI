Step 2.1.2 — Upload Domain Model
Goal

Define what an uploaded document looks like before it enters the ingestion pipeline.

This is not the SQLAlchemy model.

This is not the API schema.

This is the AI domain model.

 Upload Result

The Upload Service contains the business workflow.

It does not contain:

AWS SDK logic
SQLAlchemy
FastAPI
File parsing
Chunking
Embeddings
Responsibilities

The Upload Service should:

Accept an upload request
Validate the upload
Generate a document ID
Generate the S3 object key
Upload the file to S3
Calculate checksum
Create upload metadata
Persist metadata
Return the uploaded document

Step 2.1.5.3
Checksum Utility

Every uploaded document should receive a SHA-256 checksum.

This gives us:

Duplicate detection
Integrity verification
Future caching
Idempotency support

We'll implement a reusable checksum helper.

Step 2.1.5.4
Storage Key Generator

We need a single place responsible for object keys.

Example:

documents/{user_id}/{document_id}/original.pdf

No other class generates keys.

Upload Validation
Goal

Validate uploaded files before any storage or database operations.

Validation order:

HTTP Request
      │
      ▼
Upload Validator
      │
      ▼
Upload Service
      │
      ▼
S3
      │
      ▼
Database

Responsibilities

The validator is responsible for:

File extension
MIME type
Empty file detection
Maximum file size
Filename validation

Not responsible for:

Duplicate detection (checksum)
Virus scanning
OCR validation
PDF parsing
Document quality

Those belong to later milestones.
