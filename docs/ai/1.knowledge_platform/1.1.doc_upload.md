Step 2.1.2 — Upload Domain Model
Goal

Define what an uploaded document looks like before it enters the ingestion pipeline.

This is not the SQLAlchemy model.

This is not the API schema.

This is the AI domain model.

Upload Lifecycle

This model represents the upload during its lifecycle.

User Uploads File
        │
        ▼
Upload Domain Model
        │
        ▼
Validation
        │
        ▼
Storage
        │
        ▼
Database
        │
        ▼
Document Intelligence (2.2)

Notice the upload model exists before persistence.

Upload
    ↓
Validation
    ↓
S3 Storage
    ↓
Document Metadata (PostgreSQL)
    ↓
Document Intelligence Pipeline

The actual file lives in S3.
PostgreSQL stores only metadata and references.

Architecture
                Upload Service
                      │
                      ▼
          DocumentStorage Interface
                      │
        ┌─────────────┘
        ▼
          S3StorageService
                      │
                      ▼
                Amazon S3

The upload service only knows about DocumentStorage.

It never knows about S3.
