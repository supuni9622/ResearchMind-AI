Create a Dedicated S3 Package

Instead of placing the implementation in knowledge/upload/storage.py, I'd move AWS infrastructure into its own package.

app/
└── ai/
    ├── infrastructure/
    │
    └── storage/
        ├── __init__.py
        ├── interfaces.py
        ├── s3.py
        ├── factory.py
        └── exceptions.py

This is an intentional refinement I'd make now because storage won't only be used by uploads. Future components such as document processing, evaluation datasets, generated reports, and exports will all use it.

Then the upload module simply depends on the storage package.

S3 Client Factory

We'll build a reusable client.

class S3StorageService(DocumentStorage):
    ...

Internally it owns a boto3 client.

That client is reused.

Not recreated every upload.

Object Keys

Let's standardize the S3 layout from day one.

documents/

    {user_id}/

        {document_id}/

            original.pdf

Example:

documents/

    5ef9d...

        b1c82...

            research-paper.pdf

Later we'll add:

documents/
processed/
chunks/
reports/
exports/
datasets/

Everything remains organized.

Upload Implementation

The service should expose:

await storage.upload(...)

Internally it calls

boto3.client("s3").upload_fileobj(...)
Download
await storage.download(...)

Returns

bytes

The caller decides what to do.

Exists

Uses

head_object()

No download required.

Delete

Uses

delete_object()
Presigned URLs

We'll support

generate_presigned_url()

from day one.

We'll use them later for:

downloading reports
previewing PDFs
frontend document access

without proxying files through FastAPI.

Logging

Every storage operation logs:

bucket
key
duration
success/failure
request id

using our existing structlog configuration.

Error Handling

Create dedicated exceptions.

StorageException

├── StorageNotFound
├── StorageUploadError
├── StorageDownloadError
└── StorageDeleteError

The rest of the application never catches botocore exceptions directly.
