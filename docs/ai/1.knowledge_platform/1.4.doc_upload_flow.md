The upload service must not leave us in this state:

S3 Upload ✅
Database Insert ❌

because then we have an orphaned object in S3.

Likewise, we must avoid:

Database Insert ✅
S3 Upload ❌

This is the classic distributed transaction problem.

Since S3 and PostgreSQL cannot participate in the same ACID transaction, we'll use the Compensating Transaction (Saga) pattern.

Upload Flow
Validate
    │
    ▼
Calculate Checksum
    │
    ▼
Generate Storage Key
    │
    ▼
Upload to S3
    │
    ▼
Create Document
    │
    ▼
Persist Metadata
    │
    ├──────────────► Success ✅
    │
    ▼
Database Failure
    │
    ▼
Delete S3 Object (Compensation)
    │
    ▼
Raise Exception

This is production-grade behavior and avoids leaving orphaned files.
Let's freeze this pattern because we'll reuse it later with Qdrant, LangSmith, evaluation artifacts, etc.
