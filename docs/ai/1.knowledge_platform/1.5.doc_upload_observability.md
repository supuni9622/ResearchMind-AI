Goal

Every upload should answer four questions:

What happened? (Structured logs)
How healthy is it? (Metrics)
How long did it take? (Latency)
Can we trace it? (Correlation)
Scope (for this milestone)

We'll implement lightweight observability using the tooling we already have.

✅ Structured logging (structlog)
✅ Upload duration
✅ S3 operation duration
✅ Success/failure logs
✅ Correlation via request ID (using existing middleware)
✅ Foundation for Prometheus/OpenTelemetry later

We will not introduce OpenTelemetry or Prometheus yet. Those mature later, but the instrumentation points start now.

└── infrastructure/
    └── metrics/

Because later we'll add:

Retrieval metrics

6. Metrics We'll Track

Even though they're currently no-op, define them now.

Metric	Purpose
upload.duration	End-to-end upload latency
upload.success	Successful uploads
upload.failure	Failed uploads
storage.s3.upload.duration	S3 latency

Later these map directly to Prometheus.
