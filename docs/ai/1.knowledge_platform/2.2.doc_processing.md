# Processing Decision

**Date:** 2026-07-01

## Docling Version

**Version:** 2.107.0

## Problem

During document parsing, Docling attempted to initialize the default PDF OCR pipeline (RapidOCR / PP-OCRv6). This resulted in a runtime failure because the required OCR model path was not supported in the current environment.

## Resolution

Explicitly disable OCR when configuring the PDF pipeline.

```python
from docling.datamodel.base_models import InputFormat
from docling.datamodel.pipeline_options import PdfPipelineOptions
from docling.document_converter import (
    DocumentConverter,
    PdfFormatOption,
)

pipeline_options = PdfPipelineOptions(do_ocr=False)

converter = DocumentConverter(
    format_options={
        InputFormat.PDF: PdfFormatOption(
            pipeline_options=pipeline_options,
        ),
    }
)
```

## Reason

ResearchMind is primarily designed to process research papers from sources such as:

- arXiv
- IEEE
- ACM
- Springer
- Elsevier

These documents are almost always digitally generated PDFs containing embedded text. Running OCR on every document is unnecessary and introduces additional latency, memory usage, and external model dependencies.

Disabling OCR improves processing speed while avoiding unnecessary failures.

## Future

Introduce automatic document classification before parsing.

```
PDF
 │
 ▼
Is Scanned?
 ├── Yes → Enable OCR Pipeline
 └── No  → Standard Text Pipeline
```

OCR will be enabled only for scanned or image-based PDFs, while digital PDFs will continue to use the standard parsing pipeline.
